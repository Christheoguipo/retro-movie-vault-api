Retro Movie Vault API Documentation

This API provides a list of information about classic movies, directors and genres.

Description URL Request Query Parameters Request Body Data Format Response Format
Returns a list of ALL movies. /movies GET None. None. A JSON object holding data about all the movies. Example: 
[
  {
    "Genre": {
        "Name": "Animated",
        "Description": "Animation is a method in which pictures are manipulated to appear 
        as moving images. In traditional animation, images are drawn or painted by hand on 
        transparent celluloid sheets to be photographed and exhibited on film."
    },
    "Director": {
        "Name": "Jonathan Demme",
        "Bio": "Robert Jonathan Demme was an American director, producer, and screenwriter.",
        "Birthyear": "1944-01-01",
        "Deathyear": "2017-01-01"
    },
    "Actors": [],
    "_id": "6512158099eb8a7d7fdf59c7",
    "Title": "Silence of the Lambs",
    "Description": "A young FBI cadet must receive the help of an incarcerated and 
    manipulative cannibal killer to help catch another serial killer.",
    "Imageurl": "silenceofthelambs.png",
    "Featured": true
  }
]
Returns data about a single movie by title. /movies/:Title GET Title - The title of the movie. None. A JSON object holding data about a single movie, containing a title, description, genre, director, image url and featured. Example: 
{
  "Genre": {
      "Name": "Drama",
      "Description": "Dramas follow a clearly defined narrative plot structure, 
      portraying real-life scenarios or extreme situations with emotionally-driven characters."
  },
  "Director": {
      "Name": "James Cameron",
      "Bio": "James Cameron is a visionary director and filmmaker celebrated for his 
      groundbreaking work on movies like "Avatar" and "Titanic."",
      "Birthyear": "1954-08-16",
      "Deathyear": null
  },
  "Actors": [],
  "_id": "651218a699eb8a7d7fdf59c9",
  "Title": "Titanic",
  "Description": "PIncorporating both historical and fictionalized aspects, it is based on 
  accounts of the sinking of RMS Titanic in 1912. Kate Winslet and Leonardo DiCaprio star as 
  members of different social classes who fall in love during the ship's maiden voyage.",
  "Imageurl": "https://upload.wikimedia.org/wikipedia/en/1/18/Titanic_%281997_film%29_poster.png",
  "Featured": false
}
Returns data about a genre by genre name. /movies/genre/:GenreName GET GenreName - The genre name. None. A JSON object holding data about a genre, containing the genre name and description. Example: 
{
  "Name": "Drama",
  "Description": "Dramas follow a clearly defined narrative plot structure, portraying 
  real-life scenarios or extreme situations with emotionally-driven characters."
}
Returns data about a director by name. /directors/:Name GET Name - The name of the director. None. A JSON object holding data about a director, containing bio, birth year and death year. Example: 
{
  "Name": "Steven Spielberg",
  "Bio": "Steven Spielberg is a prolific filmmaker known for directing classic 
  movies like "E.T. the Extra-Terrestrial" and "Jurassic Park."",
  "Birthyear": "1946-12-18",
  "Deathyear": null
}
Allow new users to register. /users POST None. A JSON object holding data about a new user to register, structured like: 
{ 
  "Username": "NewUser",
  "Password": "NewUser",
  "Email": "NewUser@gmail.com",
  "Birthday": "2000-01-01"
}
A JSON object holding data about the newly registered user with auto-generated ID. Example: 
{
  "Username": "NewUser",
  "Password": "NewUser",
  "Email": "NewUser@gmail.com",
  "Birthday": "2000-01-01T00:00:00.000Z",
  "FavoriteMovies": [],
  "_id": "651367493c20572a6fe1cee6",
  "__v": 0
}
Returns a list of ALL users. /users GET None. None. A JSON object holding data about all the users. Example: 
[
  {
    "_id": "65121bbb99eb8a7d7fdf59d2",
    "Username": "user2",
    "Password": "user2",
    "Email": "user2@gmail.com",
    "Birthday": "1981-02-02T00:00:00.000Z",
    "FavoriteMovies": [
        "651219ba99eb8a7d7fdf59cf"
    ]
  }
]
Returns data about a single user by username. /users/:Username GET Username - The Username of the user. None. A JSON object holding data about a single user, containing the _id, Username, Password, Email, Birthday and FavoriteMovies. Example: 
{
  "_id": "651367493c20572a6fe1cee6",
  "Username": "NewUser",
  "Password": "NewUser",
  "Email": "NewUser@gmail.com",
  "Birthday": "2000-01-01T00:00:00.000Z",
  "FavoriteMovies": [],
  "__v": 0
}
Allow users to update their user info. /users/:Username PUT Username - The Username of the user to be updated. A JSON object holding data about the user's changes on his/her name, structured like: 
{
  "Username": "NewUser",
  "Password": "NewPassword",
  "Email": "NewEmail@gmail.com",
  "Birthday": "1990-12-12"
}
A JSON object holding data about the user's applied changes on his/her name. Example: 
{
  "_id": "651367493c20572a6fe1cee6",
  "Username": "NewUser",
  "Password": "NewPassword",
  "Email": "NewEmail@gmail.com",
  "Birthday": "1990-12-12T00:00:00.000Z",
  "FavoriteMovies": [],
  "__v": 0
}
Allow users to add a movie to their list of favorites. /users/:Username/movies/:MovieID POST Username - The Username of the user.
MovieID - The MovieID of the movie which will be added to their list of favorites. 		None. A JSON object holding the updated data of the user where the MovieID is added to the FavoriteMovies array. Example: 
{
  "_id": "65134b87177fc702f561b6c5",
  "Username": "User1",
  "Password": "1111",
  "Email": "user3@gmail.com",
  "FavoriteMovies": [
      "6512158099eb8a7d7fdf59c7"
  ],
  "__v": 0
}
Allow users to remove a movie from their list of favorites. /users/:Username/movies/:MovieID DELETE Username - The Username of the user.
MovieID - The MovieID of the movie which will be removed from their list of favorites. 		None. A JSON object holding the updated data of the user where the MovieID is removed from the FavoriteMovies array. Example: 
{
  "_id": "65134b87177fc702f561b6c5",
  "Username": "User1",
  "Password": "1111",
  "Email": "user3@gmail.com",
  "FavoriteMovies": [],
  "__v": 0
}
Allow existing users to deregister. /users/:Username DELETE Username - The Username of the user to deregister. None. A text message indicating whether the user was successfully removed.
