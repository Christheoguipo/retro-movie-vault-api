Retro Movie Vault API Documentation

This API provides a list of information about classic movies, directors and genres.

Description URL Request Query Parameters Request Body Data Format Response Format
Returns a list of ALL movies. /movies GET None. None. A JSON object holding data about all the movies. Example: 
[
  {
    "title": "Movie Title",
    "description": "Movie Description.",
    "genre": {
      "name": "Crime",
      "description": "Crime genre is largely classified by a story that is 
              centered around the solving of a crime." 
    },
    "director": { 
      "name": "Quentin Tarantino",
      "bio": "His Bio.",
      "birthyear": 1963, 
      "deathyear": null
    },
    "image_url": "https://website.com/Image.jpg",
    "featured": false
  },
]
Returns data about a single movie by title. /movies/:title GET title - The title of the movie. None. A JSON object holding data about a single movie, containing a title, description, genre, director, image url and featured. Example: 
{
  "title": "Movie Title",
  "description": "Movie Description.",
  "genre": {
    "name": "Crime",
    "description": "Crime genre is largely classified by a story that is 
            centered around the solving of a crime." 
  },
  "director": { 
    "name": "Quentin Tarantino",
    "bio": "His Bio.",
    "birthyear": 1963, 
    "deathyear": null
  },
  "image_url": "https://website.com/Image.jpg",
  "featured": false
}
Returns data about a genre by genre name. /movies/genre/:genreName GET genreName - The genre name. None. A JSON object holding data about a genre, containing the genre name and description. Example: 
{
  "name": "Crime",
  "description": "Crime genre is largely classified by a story that is 
          centered around the solving of a crime."
}
Returns data about a director by name. /directors/:name GET name - The name of the director. None. A JSON object holding data about a director, containing bio, birth year and death year. Example: 
{
  "name": "Quentin Tarantino",
  "bio": "His Bio.",
  "birthyear": 1963, 
  "deathyear": null
}
Allow new users to register. /users POST None. A JSON object holding data about a new user to register, structured like: 
{
  "name": "User3",
  "email": "user3@gmail.com",
  "favoriteMovies": []
}
A JSON object holding data about the newly registered user with auto-generated ID. Example: 
{
  "name": "User3",
  "email": "user3@gmail.com",
  "favoriteMovies": [],
  "id": "8b4b2e2d-17ea-4d2e-8ce2-e8b462ba57fd"
}
Allow users to update their user info. /users/:id PUT id - The id of the user to be updated. A JSON object holding data about the user's changes on his/her name, structured like: 
{
  "name": "User3Updated"
}
A JSON object holding data about the user's applied changes on his/her name. Example: 
{
  "name": "User3Updated",
  "email": "user3@gmail.com",
  "favoriteMovies": [],
  "id": "8b4b2e2d-17ea-4d2e-8ce2-e8b462ba57fd"
}
Allow users to add a movie to their list of favorites. /users/:id/:movieTitle POST id - The id of the user.
movieTitle - The title of the movie which will be added to their list of favorites. 		None. A text message indicating whether the movie was successfully added to the user's favorites.
Allow users to remove a movie from their list of favorites. /users/:id/:movieTitle DELETE id - The id of the user.
title - The title of the movie. 		None. A text message indicating whether the movie was successfully deleted from the user's favorites.
Allow existing users to deregister. /users/:id DELETE id - The id of the user to deregister. None. A text message indicating whether the user's email was successfully removed.
